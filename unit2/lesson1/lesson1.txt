4. Virtual DOM => react's superpower
what's is DOM => document object model => browser's representation of UI tree => upadating it directly(via vanilla js) is slow.

what react does ?
1. react maintains a Virtual DOM(a lightweight JS copy of real DOM)
2. when state or props change -> react creates a new Virtual DOM.
3. react compares new Virtual DOM with old one (diffing algorithm)
4. it updates the only changed elements in the real DOM.

example(conceptual)
old UI: <p>Sunny</p>
new UI: <P>Rahul</p>
=> react detects the only the text node changed not the <p> tag and updates just that part in real DOM => that's why react feels instant and smooth

5. props => passing data between components
props => properties =>  inputs to a component => they make components dynammic and reusable

..Greeting.jsx..
function Greeting(props) {
  return <h2>Hello, {props.name}!</h2>;
}

export default Greeting;

..App.jsx..
import Greeting from './Greeting';

function App() {
  return (
    <div>
      <Greeting name="Sunny" />
      <Greeting name="Riya" />
      <Greeting name="Karan" />
    </div>
  );
}

export default App;

example 2: destructuring props(cleaner syntax)

..Greeting.jsx..

..main.jsx..
